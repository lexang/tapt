# 表格转换工具站重构计划

## 目标

把当前 MVP 工具页升级为更像正式产品的中文在线表格转换工具站。优先改善用户真正会感知的部分：上传、转换选项、表格编辑、三栏工作台、响应式体验、错误反馈和视觉质量。

## 阶段

- [complete] 阶段 1：梳理现有工作台、表格编辑器、样式和测试边界
- [complete] 阶段 2：重构转换状态模型，加入文件上传、错误状态和转换选项
- [complete] 阶段 3：升级工作台 UI，形成输入、预览、输出和选项的专业工具布局
- [complete] 阶段 4：增强表格编辑器，支持改表头、增删行列、清空和空状态
- [complete] 阶段 5：补充测试并运行单元、构建、端到端验证
- [complete] 阶段 6：浏览器截图审查和最终提交

## 设计原则

- 用户数据默认在浏览器中处理。
- 页面文案必须是用户视角，不能出现内部实现或阶段说明口吻。
- 移动端优先可用，桌面端提供高密度三栏工作台。
- 重构保持 Next.js App Router 和现有解析/生成器架构。

## 风险

- Excel 解析/生成依赖 `xlsx`，需要控制导入边界，避免无谓增加首页负担。
- 文件上传和文本输入共用状态后，错误提示必须清晰，不能吞掉用户输入。
- 表格编辑器如果直接用 `contentEditable`，受控状态和光标行为容易不稳定。

## 验收结果

- `npm test`：15 个测试文件，26 个测试通过。
- `npm run build`：通过，首屏 JS 约 113 kB。
- `npx playwright test`：5 个端到端测试通过。
