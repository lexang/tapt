# TableConvert 竞品调研记录

调研对象：https://tableconvert.com/zh-cn/

调研日期：2026-05-15

## 一句话结论

TableConvert 不是单一格式转换器，而是围绕“表格数据输入、在线编辑、格式生成、SEO 长尾页面”构建的一套在线工具矩阵。它的核心价值是让用户把 Excel、CSV、JSON、Markdown、SQL、HTML 等表格或结构化数据快速互转，并在浏览器内完成编辑、复制和下载。

## 产品定位

- 免费在线表格转换器和表格生成器。
- 面向开发者、办公用户、数据分析人员、学生和研究人员。
- 强调浏览器本地处理、隐私安全、无需安装。
- 通过大量格式组合页面获取搜索流量。

## 核心用户流程

1. 用户进入某个转换页面，例如 Excel 转 JSON。
2. 选择或确认数据源格式。
3. 粘贴数据、上传文件、使用示例数据，或从网页提取表格。
4. 在在线表格编辑器中预览和清洗数据。
5. 选择输出格式和输出选项。
6. 复制结果或下载文件。

## 页面结构观察

### 顶部导航

- 品牌入口：TableConvert。
- 版本入口：v3.0.1 / 更新日志。
- 搜索框：搜索转换器。
- 快捷入口：CSV 转 Excel、LaTeX 表格生成器、扩展程序、API 文档、Blog。
- 工具按钮：深色模式、语言切换、赞助入口。

### 主工具区

主工具区按三段组织：

- 数据源
- 在线表格编辑器
- 表格生成器

这三个区块构成页面的主要使用路径，用户进入页面后不需要先读营销内容，可以直接开始转换。

### 数据源区

支持能力：

- 选择输入格式，例如 Excel。
- 加载示例数据。
- 上传文件。
- 从网页提取表格。
- 粘贴数据或拖拽文件。
- 展示当前格式支持的文件扩展名，例如 `.xls`、`.xlsx`、`.xlsm`。

### 在线表格编辑器

观察到的编辑能力：

- 调整表格尺寸，例如 10 x 10。
- 展开 / 收起。
- 全屏编辑。
- 撤销。
- 重做。
- 转置行列。
- 清空。
- 删除空行或空列。
- 删除重复行。
- 转大写。
- 转小写。
- 首字母大写。
- 查找替换。
- 支持正则表达式查找替换。

### 表格生成器

输出区能力：

- 选择输出格式。
- 复制到剪贴板。
- 下载文件。
- 输出结果文本框。
- 根据输出格式提供专属选项。

以 Excel 转 JSON 为例，选项包括：

- 解析单元格内 JSON 字符串。
- 压缩输出。
- 数据格式：Array of Object、2D Array、Column Array、Keyed Array。
- 根对象名称。
- 缩进大小：2 spaces、4 spaces、8 spaces、Tabs。

## SEO 策略

TableConvert 的页面矩阵非常明显：

- `/zh-cn/excel-to-json`
- `/zh-cn/csv-to-json`
- `/zh-cn/json-to-excel`
- `/zh-cn/excel-to-sql`
- `/zh-cn/markdown-to-excel`
- `/zh-cn/html-to-json`

每个转换组合都有独立 URL、独立标题、说明、步骤、FAQ 和相关工具推荐。这是它获取搜索流量的关键。

可借鉴策略：

- 为每个高频转换组合生成独立页面。
- 页面主标题直接匹配搜索词，例如“Excel 转 JSON”。
- 工具本体放在首屏或接近首屏。
- 页面下方放使用步骤、FAQ、相关转换器。
- 文案必须站在用户视角，避免产品内部说明口吻。

观察到的入口：

- API 文档。
- Blog。

第一版可暂缓 API、浏览器插件和赞助商，但页面结构可以预留资源入口。

## 视觉与交互特点

- 页面不是传统营销首页，而是工具优先。
- 主体是密集但清晰的工作台界面。
- 使用浅色背景、卡片式工具区、紧凑按钮和标签。
- 支持深色模式。
- 格式导航以大量标签链接呈现。
- 输出选项与当前输出格式强绑定。
- 复制、下载是最核心的结果操作。

## 可借鉴点

- 工具即首页，减少用户到达工具前的步骤。
- 输入、编辑、输出三段式结构清晰。
- 每个转换组合独立页面，适合 SEO。
- 所有转换尽量在浏览器本地完成，提升隐私信任。
- 高频输出选项要做细，例如 JSON 缩进、压缩、数组结构。
- 编辑器需要提供常见数据清洗动作，而不仅是预览。

## 需要避免的问题

- 不要第一版堆太多格式，容易导致质量不稳定。
- 不要复制 TableConvert 的页面文案和品牌表达。
- 不要把“第一版、后续、用于承接”等内部产品说明写到用户页面。
- 不要让首页变成营销页，用户来这里是为了马上转换数据。
- 不要让格式矩阵变成空链接，首批页面必须真实可用。

## 差异化方向

可考虑从中文体验和简洁工作台切入：

- 中文文案更自然，不写机翻式说明。
- 转换路径更短，减少干扰内容。
- 输出选项更聚焦常见场景。
- 移动端可查看，但主要优化桌面端效率。
- 首屏强调“数据在本地浏览器处理”。
