# 中文表格转换工具站第一版设计

日期：2026-05-15

## 背景

目标是做一个类似 TableConvert 的中文在线表格转换工具站，但第一版不完整复刻所有格式和增长模块。第一版优先解决高频转换场景，让用户进入页面后可以直接粘贴、上传、编辑、转换、复制或下载数据。

竞品调研记录见：`docs/research/tableconvert-竞品调研.md`。

## 产品定位

中文用户优先的免费在线表格转换工具站。

核心承诺：

- 打开即可转换表格数据。
- 常用格式之间互转稳定可靠。
- 数据在浏览器本地处理，不上传到服务器。
- 转换结果可以直接复制或下载。

第一版面向：

- 需要把 Excel 或 CSV 转成 JSON、SQL、Markdown 的开发者。
- 需要把表格转成文档格式的内容编辑者。
- 需要快速清洗表格数据的办公用户。
- 需要临时处理结构化数据的学生、运营和数据分析人员。

## 第一版范围

### 支持格式

输入格式：

- Excel
- CSV
- JSON
- Markdown
- SQL
- HTML

输出格式：

- Excel
- CSV
- JSON
- Markdown
- SQL
- HTML

第一版优先保证以下转换路径：

- Excel 转 JSON
- Excel 转 CSV
- Excel 转 Markdown
- Excel 转 SQL
- CSV 转 JSON
- CSV 转 Markdown
- CSV 转 SQL
- JSON 转 CSV
- JSON 转 Excel
- JSON 转 Markdown
- Markdown 转 CSV
- Markdown 转 JSON
- HTML 表格转 CSV
- HTML 表格转 JSON

### 暂不包含

- 登录和用户系统。
- 云端保存历史。
- 在线协作。
- 付费订阅。
- 浏览器扩展。
- 公开 API。
- 超过首批格式以外的长尾转换。
- 后台管理系统。

这些能力不进入第一版，但技术结构应避免阻碍后续扩展。

## 信息架构

### 首页

首页直接呈现转换工作台，而不是营销落地页。

首屏结构：

- 顶部导航：品牌、转换器搜索、热门转换入口、深色模式、语言入口。
- 工具标题：根据当前转换路径展示，例如“Excel 转 JSON”。
- 隐私提示：简洁说明数据在本地浏览器处理。
- 三段式工作台：数据源、表格编辑器、结果生成器。

页面下方结构：

- 使用步骤。
- 常见问题。
- 相关转换器。
- 支持格式入口。
- 页脚资源入口。

### 独立转换页面

每个高频转换组合有独立 URL，例如：

- `/zh/excel-to-json`
- `/zh/csv-to-json`
- `/zh/json-to-excel`
- `/zh/excel-to-sql`
- `/zh/markdown-to-json`
- `/zh/html-to-csv`

独立页面共享同一个工作台组件，但根据输入和输出格式调整：

- 页面标题。
- 示例数据。
- 输入提示。
- 输出选项。
- 使用步骤。
- FAQ。
- 相关转换器。

### 转换器搜索

搜索目标是帮助用户快速找到格式组合。

用户输入关键词时，匹配：

- 输入格式。
- 输出格式。
- 中文别名，例如“表格转 JSON”“Excel 转接口数据”。
- 英文格式名，例如 CSV、JSON、SQL。

## 核心用户流程

1. 用户打开首页或某个独立转换页面。
2. 页面默认展示一个高频转换组合。
3. 用户选择输入格式和输出格式。
4. 用户粘贴数据、上传文件或加载示例。
5. 系统解析为统一的表格数据模型。
6. 用户在表格编辑器中检查和清洗数据。
7. 系统根据输出格式实时生成结果。
8. 用户复制结果或下载文件。

## 页面组件设计

### 顶部导航

职责：

- 提供品牌入口。
- 提供转换器搜索。
- 提供热门工具快捷入口。
- 提供深色模式切换。
- 保留语言切换入口。

用户可见文案应保持直接，例如：

- “搜索转换器”
- “Excel 转 JSON”
- “CSV 转 Markdown”

避免使用内部说明式文案。

### 数据源区

职责：

- 展示当前输入格式。
- 支持切换输入格式。
- 支持粘贴数据。
- 支持上传文件。
- 支持加载示例数据。
- 展示支持的文件类型。

状态：

- 空状态：提示用户粘贴或上传数据。
- 解析中：提示正在读取文件。
- 解析成功：显示解析到的行列数量。
- 解析失败：用中文说明失败原因和可尝试操作。

### 表格编辑器

职责：

- 展示解析后的表格。
- 支持直接编辑单元格。
- 支持新增、删除行列。
- 支持撤销和重做。
- 支持转置。
- 支持删除空行。
- 支持去重。
- 支持查找替换。

第一版重点做“够用且稳定”的编辑能力，不追求完整电子表格能力。

### 结果生成器

职责：

- 展示输出格式。
- 根据输出格式显示对应选项。
- 实时生成结果。
- 支持复制。
- 支持下载。
- 支持错误提示。

JSON 输出选项：

- 美化输出。
- 压缩输出。
- 缩进大小。
- 输出结构：对象数组、二维数组。

SQL 输出选项：

- 表名。
- INSERT 语句。
- 是否包含列名。
- 字符串转义。

Markdown 输出选项：

- 是否包含表头。
- 是否对齐列。

HTML 输出选项：

- 只输出 table。
- 是否包含 thead 和 tbody。
- 是否美化缩进。

CSV 输出选项：

- 分隔符。
- 是否包含表头。
- 换行符。

Excel 输出选项：

- 文件名。
- 工作表名称。

## 数据模型

所有输入先解析为统一的中间表格模型：

```ts
type TableCell = {
  value: string;
};

type TableData = {
  columns: string[];
  rows: TableCell[][];
};
```

转换流程：

```text
输入内容或文件
  -> 输入解析器
  -> TableData
  -> 表格编辑器
  -> 输出生成器
  -> 文本结果或文件结果
```

这种结构可以让每个输入格式和输出格式独立开发，避免转换器之间互相耦合。

## 技术架构建议

推荐使用 Next.js + React + TypeScript。

理由：

- 适合生成大量 SEO 独立页面。
- React 适合构建复杂工作台交互。
- TypeScript 能降低格式转换逻辑的维护风险。
- 后续可扩展 API、批处理和浏览器扩展。

核心模块：

- `app`：页面路由和 SEO 页面。
- `components/workbench`：转换工作台组件。
- `components/table-editor`：表格编辑器组件。
- `lib/parsers`：输入解析器。
- `lib/generators`：输出生成器。
- `lib/table`：统一表格模型和清洗操作。
- `lib/converters`：格式组合配置。
- `content/converters`：每个转换页面的标题、说明、FAQ 和示例。

文件解析可使用成熟库：

- Excel：`xlsx`
- CSV：`papaparse`
- Markdown 表格：自定义轻量解析，或基于统一规则解析。
- HTML 表格：浏览器 DOMParser。
- JSON：原生 JSON.parse 加结构推断。
- SQL：第一版优先支持常见 INSERT 语句解析。

## 错误处理

错误信息使用中文，并给出用户能执行的下一步。

示例：

- “没有识别到表格数据，请检查内容是否包含表头和数据行。”
- “JSON 格式不完整，请检查括号、逗号和引号是否正确。”
- “文件读取失败，请确认文件没有损坏，并重新上传。”
- “当前内容为空，粘贴数据后即可生成结果。”

## SEO 与内容策略

每个独立转换页面包含：

- 唯一标题。
- 简短说明。
- 可直接使用的转换工具。
- 三步使用说明。
- 常见问题。
- 相关转换器链接。

页面文案必须回答用户关心的问题：

- 能转什么。
- 怎么转。
- 数据是否安全。
- 结果能复制还是下载。
- 输出格式适合什么场景。

避免在前台出现：

- “第一版”
- “后续再”
- “当前阶段”
- “用于承接”
- “管理员发布”
- “普通用户”

## 测试策略

### 单元测试

覆盖：

- CSV 解析和生成。
- JSON 解析和生成。
- Markdown 表格解析和生成。
- HTML 表格解析和生成。
- SQL 生成。
- 表格转置、去重、删除空行。

### 组件测试

覆盖：

- 粘贴数据后表格更新。
- 切换输出格式后结果更新。
- 修改单元格后结果实时变化。
- 复制和下载按钮状态。

### 端到端测试

覆盖核心路径：

- CSV 转 JSON。
- Excel 转 JSON。
- JSON 转 CSV。
- Markdown 转 HTML。

## 验收标准

第一版完成时，应满足：

- 用户可以在首页完成一次 CSV 转 JSON。
- 用户可以上传 Excel 并生成 JSON。
- 用户可以编辑表格数据并看到输出实时更新。
- 用户可以复制输出结果。
- 用户可以下载输出文件。
- 高频转换页面有独立 URL 和基础 SEO 内容。
- 页面文案为中文用户视角。
- 常见解析错误有明确中文提示。
- 核心转换逻辑有自动化测试。

## 后续扩展方向

这些方向不进入第一版实现，但架构上应保留空间：

- 浏览器扩展，从网页提取表格。
- 批量文件转换。
- API 服务。
- 更多格式，例如 YAML、TOML、PDF、图片。
- 转换历史，仅本地存储。
- 多语言页面。
